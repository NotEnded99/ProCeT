## 2. CSR方案的优势（针对大规模子区域）

### 2.1 计算效率与区域数量无关

CSR方案的**核心优势**：
- **协方差矩阵计算是统计平均**：即使有300万个区域，只需计算O(d²)运算
- **子空间分解是O(d³)**：与d维状态空间有关，与区域数量无关！

**时间复杂度对比**：
| 操作 | 时间复杂度 | 300万个区域的实际时间 |
|------|-----------|-------------------|
| 增量协方差更新 | 每个区域O(d²) | < 10秒 |
| 广义特征值分解 | O(d³) | < 1秒 |
| 总计 | O(Nd² + d³) | < 20秒 |

**结论**：即使有300万个区域，CSR分析也只需20秒！

### 3.1 增量协方差计算

```python
class IncrementalCovariance:
    def __init__(self, d: int):
        self.n_V = 0
        self.M_V = np.zeros((d, d))
        self.M_F = np.zeros((d, d))

    def update_verified(self, A_L):
        self.n_V += 1
        delta = (A_L.T @ A_L - self.M_V) / self.n_V
        self.M_V += delta
```

**优势**：
- 不需要存储所有A_L矩阵（节省300万个*d的内存）
- 可以边验证边计算，不需要等验证完成

### 3.2 统计采样策略

## 5. 为什么CSR方案特别适合这种场景？

### 5.1 与区域划分的天然契合

论文中的验证流程（图1）：
1. 将空间划分为m个单纯形
2. 对每个单纯形验证，提取A_L矩阵
3. 如果失败且体积>阈值，细分该单纯形

**CSR可以完美集成到这个流程中**：
- 在步骤2中，边验证边增量收集A_L
- 验证完成后，立即执行CSR分析
- 只需要修改几行代码！

### 5.2 统计特性的利用

虽然区域数量巨大，但它们的A_L矩阵**统计上聚类**：
- 已验证区域的A_L指向相似方向（网络对这些区域有共识）
- 失败区域的A_L指向另一个方向
- 采样1%的区域就足够了！
